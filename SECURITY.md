# Security Policy

## Reporting a Vulnerability

OpenSend is a side project with no dedicated security team. If you find a security issue:

1. **Do not** open a public GitHub issue
2. Email: sparshsam@gmail.com
3. Include as much detail as possible:
   - Type of vulnerability
   - Steps to reproduce
   - Potential impact

## Response Timeline

- Initial acknowledgment: within 72 hours
- Fix timeline: depends on severity, typically 1-14 days
- Public disclosure: after a fix is deployed

## Scope

- `opensend.vercel.app` (production deployment)
- The MCP server (`apps/mcp/`)
- Supabase database and storage configurations

## Out of Scope

- Third-party services (Supabase, Vercel, GitHub)
- Self-hosted instances (file issues on your own fork)
